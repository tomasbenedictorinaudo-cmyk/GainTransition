1. Purpose & Scope

This document specifies the G4-Compensated (After) algorithm for transitioning a satellite payload from one gain configuration to another. The algorithm uses gain stage G4 (a per-channel digital gain on the transmit side) as a real-time compensation lever: after each primary gain step, it adjusts G4 on all affected channels to bring the EIRP back toward the initial operating point.

Scope: This document covers only the g4-compensated strategy with mode = "after". It is self-contained: an external team should be able to implement the complete algorithm from this document alone, with no access to other source code.

Why G4? G4 is a per-channel digital gain stage. It can be independently adjusted for each channel without affecting any other channel. This makes it ideal as a compensation lever: when a shared gain (e.g., G1 on an Rx antenna) changes and shifts the EIRP of multiple channels, G4 can independently correct each affected channel.

2. System Context

2.1 Payload Signal Chain

Each channel passes through 7 gain stages in series:

2.2 Stage Classification

CategoryStagesMove Rule
AnalogG1, G7One antenna per iteration. Only one gain instance changes.

Digital (per-channel)G3, G4, G5All instances of the stage type move together in one iteration.

2.3 Channel Gain Chain

For a channel c with Rx antenna r, Tx antenna t, Rx primary subchannel srx, and Tx primary subchannel stx, the ordered gain chain is:

[G1:rxr,   G2:rxr:subsrx,   G3:chc,   G4:chc,   G5:chc,   G6:txt:substx,   G7:txt]

The primary subchannel index is computed as: subchannelIndex = floor(lowFreqMHz / 5)

4. Algorithm Overview

REQ-001 The algorithm SHALL operate in two sequential phases: Phase 1 (non-G4 gains with G4 compensation) followed by Phase 2 (G4 gains to final targets).
REQ-002 In Phase 1, after each primary gain step, the algorithm SHALL insert a separate G4 correction step that adjusts G4 on all affected channels to minimize cumulative EIRP deviation from the initial state.
REQ-003 In Phase 2, G4 gains SHALL be stepped to their final target values. This phase accepts that EIRP deviation will occur as the compensation cushion is removed.
REQ-004 Each iteration (in either phase) SHALL change only one gain stage TYPE (e.g., all G3 instances, or one G1 antenna).
REQ-005 For analog stages (G1, G7), each iteration SHALL change the gain of exactly one antenna.
REQ-006 For digital stages (G2, G3, G4, G5, G6), each iteration SHALL move all instances of that stage type together.
REQ-007 A gain stage MAY be changed by more than one granularity step in a single iteration (1·γ, 2·γ, …, n·γ where n·γ ≤ remaining delta).

5. Phase 1: Non-G4 Stepping with G4 Compensation

5.1 Requirements

REQ-010 Phase 1 SHALL iterate until all non-G4 gain stages have reached their target values, or until no feasible move exists, or until the iteration limit is reached.
REQ-011 In Phase 1, candidate generation SHALL exclude all G4 gain stages.
REQ-012 After selecting and applying the primary move, the algorithm SHALL compute a G4 correction step that adjusts G4 for every channel whose cumulative EIRP deviates from the initial EIRP.
REQ-013 The G4 correction step SHALL be recorded as a separate TransitionStep (not merged with the primary step).
REQ-014 Multiple G4 gain instances SHALL be updated simultaneously in a single G4 correction step.

5.2 Pseudocode

function runPhase1(channels, gainStages, params):
    {gainValues, targetValues, granularities} = initState(gainStages)
    initialEirp = computeAllChannelEirp(channels, gainValues)
    initialSystemTemp = computeAllSystemTemp(channels, gainValues, gainStages)
    steps = []
    violations = 0
    hasEirpLimits = (params.maxNegativeEirpDeviation != null
                  OR params.maxPositiveEirpDeviation != null)

    for iter = 0 to params.maxIterations - 1:

        // Check if all non-G4 gains are at target
        nonG4Done = true
        for each key in targetValues:
            if key starts with "G4:": continue
            if |gainValues[key] - targetValues[key]| >= granularities[key] * 0.01:
                nonG4Done = false
                break
        if nonG4Done: break

        // 1. Generate candidates excluding G4
        candidates = generateCandidateMoves(gainValues, targetValues,
                                            granularities, excludeTypes={"G4"})
        if candidates is empty: break

        // 2. Filter by power thresholds
        feasible = filter candidates where:
            checkPowerFeasibility(move, gainValues, channels, gainStages) == true

        if feasible is empty:
            // Deadlock: relax power thresholds by 1 dB
            feasible = filterRelaxedPower(candidates, gainValues, channels, gainStages)
            if feasible is empty: break   // truly stuck
            violations++

        // 3. Filter by EIRP limits (checking BOTH intermediate states)
        if hasEirpLimits:
            eirpFiltered = filterEirpWithG4After(
                feasible, gainValues, channels, initialEirp, params,
                granularities, gainStages)
            if eirpFiltered is empty:
                return ERROR("EIRP limits infeasible",
                    bestAchievableDeviation(feasible, gainValues, channels, initialEirp))
            feasible = eirpFiltered

        // 4. Pick best candidate (most progress)
        best = pickBestByProgress(feasible)

        // 5. Apply primary move
        applyMove(best, gainValues)
        record TransitionStep(best, gainValues, channels, gainStages, initialEirp)

        // 6. Compute and apply G4 correction
        g4Correction = computeG4CorrectionAfter(
            gainValues, channels, initialEirp, granularities, gainStages)
        if g4Correction is not null:
            applyMove(g4Correction, gainValues)
            record TransitionStep(g4Correction, gainValues, channels, gainStages, initialEirp)

    return {steps, gainValues, violations}

6. Phase 2: G4 Stepping to Final Targets

6.1 Requirements

REQ-020 Phase 2 SHALL iterate until all gain stages (including G4) have reached their target values.
REQ-021 In Phase 2, candidate generation SHALL produce only G4 candidates.
REQ-022 In Phase 2, there is NO G4 correction step. Candidates are filtered by standard EIRP limits (single post-move state).

6.2 Pseudocode

function runPhase2(channels, gainStages, params, gainValues, targetValues,
                   granularities, initialEirp, steps, violations):

    hasEirpLimits = (params.maxNegativeEirpDeviation != null
                  OR params.maxPositiveEirpDeviation != null)

    for iter = 0 to params.maxIterations - 1:

        if allAtTarget(gainValues, targetValues, granularities): break

        // Generate all candidates, then keep only G4
        allCandidates = generateCandidateMoves(gainValues, targetValues, granularities)
        g4Candidates = filter allCandidates where stageType == "G4"
        if g4Candidates is empty: break

        // Filter by power thresholds
        feasible = filter g4Candidates where:
            checkPowerFeasibility(move, gainValues, channels, gainStages) == true

        if feasible is empty:
            feasible = filterRelaxedPower(g4Candidates, gainValues, channels, gainStages)
            if feasible is empty: break
            violations++

        // Filter by EIRP limits (standard single-state check)
        if hasEirpLimits:
            eirpFiltered = filterEirpStandard(
                feasible, gainValues, channels, initialEirp, params)
            if eirpFiltered is empty:
                return ERROR("EIRP limits infeasible in Phase 2",
                    bestAchievableDeviation(feasible, gainValues, channels, initialEirp))
            feasible = eirpFiltered

        // Pick best by progress
        best = pickBestByProgress(feasible)
        applyMove(best, gainValues)
        record TransitionStep(best, gainValues, channels, gainStages, initialEirp)

    return {steps, gainValues, violations}

7. Candidate Generation

REQ-030 The candidate generator SHALL group pending gain changes by stage type (G1, G2, …, G7).
REQ-031 For analog stages (G1, G7), the generator SHALL produce one candidate per antenna per step multiplier (1·γ, 2·γ, …, up to the remaining delta for that antenna).
REQ-032 For digital stages (G2–G6), the generator SHALL produce one candidate per step multiplier. Within each candidate, every instance of that stage type moves by min(multiplier, instance_remaining_steps) · γ toward its target.
REQ-033 A gain stage that has already reached its target (remaining delta < 0.01·γ) SHALL NOT produce candidates.
REQ-034 When excludeTypes is provided, all stage types in the set SHALL be excluded from candidate generation.

7.1 Pseudocode

function generateCandidateMoves(gainValues, targetValues, granularities, excludeTypes={}):
    ANALOG_STAGES = {"G1", "G7"}
    moves = []

    // Group pending changes by stage type
    pendingByType = new Map()
    for each key in targetValues:
        remaining = targetValues[key] - gainValues[key]
        gran = granularities[key]
        if |remaining| < gran * 0.01: continue    // already at target

        stageType = key.split(":")[0]              // e.g., "G1", "G3"
        if stageType in excludeTypes: continue

        direction = +1 if remaining > 0, else -1
        maxSteps = round(|remaining| / gran)
        add {key, maxSteps, gran, direction} to pendingByType[stageType]

    // Generate candidates for each type
    for each (stageType, pending) in pendingByType:

        if stageType in ANALOG_STAGES:
            // One antenna per candidate, with varying step sizes
            for each {key, maxSteps, gran, direction} in pending:
                for n = 1 to maxSteps:
                    moves.push({
                        steps: [{gainStageKey: key, delta: direction * n * gran}],
                        stageType: stageType
                    })

        else:  // Digital stage
            // All instances together, with varying multiplier
            maxMultiplier = max(p.maxSteps for p in pending)
            for n = 1 to maxMultiplier:
                steps = []
                for each {key, maxSteps, gran, direction} in pending:
                    actualN = min(n, maxSteps)
                    steps.push({gainStageKey: key, delta: direction * actualN * gran})
                moves.push({steps, stageType})

    return moves

8. Power Feasibility Check

REQ-040 Before selecting a candidate, the algorithm SHALL verify that applying the move does not cause any gain stage output power to exceed its upper or lower threshold.
REQ-041 A tolerance of 0.001 dBm SHALL be applied to avoid floating-point boundary rejection.

8.1 Pseudocode

function checkPowerFeasibility(move, gainValues, channels, gainStages):
    tempGains = copy of gainValues
    for each step in move.steps:
        tempGains[step.gainStageKey] += step.delta

    powerLevels = computeAllPowerLevels(channels, tempGains)

    for each (key, power) in powerLevels:
        stage = gainStages.get(key)
        if power > stage.upperThreshold + 0.001: return false
        if power < stage.lowerThreshold - 0.001: return false

    return true

9. EIRP Computation

REQ-050 The EIRP of channel c SHALL be computed as the sum of input power and all 7 gain stages in the channel's chain.

9.1 Formulas

Input power:

Pin,c = IPFDc + 10 · log10(BWc · 106)    [dBm]

Channel EIRP:

EIRPc = Pin,c + gG1 + gG2 + gG3 + gG4 + gG5 + gG6 + gG7    [dBm]

where each g is the current gain value (dB) of the corresponding stage in the channel's chain.

EIRP deviation from initial:

ΔEIRPc = EIRPc(current) − EIRPc(initial)    [dB]

9.2 Pseudocode

function computeChannelEirp(channel, gainValues):
    chain = getChannelGainChain(channel)   // [G1:rx.., G2:rx.., G3:ch.., G4:ch.., G5:ch.., G6:tx.., G7:tx..]
    pIn = channel.ipfd + 10 * log10(channel.bandwidthMHz * 1e6)
    eirp = pIn
    for each key in chain:
        eirp += gainValues[key]
    return eirp

function computeAllChannelEirp(channels, gainValues):
    result = {}
    for each channel in channels:
        result[channel.id] = computeChannelEirp(channel, gainValues)
    return result

10. EIRP Deviation Filtering (with Intermediate State Check)

REQ-060 In Phase 1, the EIRP filter SHALL check both intermediate states in the two-step sequence:
  1. State A (post-primary, pre-G4): EIRP after the primary move is applied but before G4 correction.
  2. State B (post-G4): EIRP after G4 correction is also applied.
A candidate is rejected if either state violates the EIRP limits.
REQ-061 In Phase 2, the EIRP filter SHALL check only the single post-move state (standard check).
REQ-062 The negative EIRP limit SHALL reject any move where any channel's deviation is below −maxNegativeEirpDeviation (with 0.001 dB tolerance).
REQ-063 The positive EIRP limit SHALL reject any move where any channel's deviation exceeds +maxPositiveEirpDeviation (with 0.001 dB tolerance).
REQ-064 Either limit may be null (unconstrained) independently of the other.

10.1 Pseudocode: Phase 1 EIRP Filter (After Mode)

function filterEirpWithG4After(candidates, gainValues, channels,
                                initialEirp, params, granularities, gainStages):
    negLimit = params.maxNegativeEirpDeviation
    posLimit = params.maxPositiveEirpDeviation

    return filter candidates where:

        // State A: after primary step (before G4)
        tempA = copy of gainValues
        for each step in move.steps:
            tempA[step.gainStageKey] += step.delta
        eirpA = computeAllChannelEirp(channels, tempA)

        if NOT checkEirpWithinLimits(eirpA, initialEirp, channels, negLimit, posLimit):
            reject

        // State B: after G4 correction
        currentEirp = computeAllChannelEirp(channels, tempA)
        g4Correction = computeG4StepsFromDeviation(initialEirp, currentEirp,
                                                    channels, tempA, granularities, gainStages)
        if g4Correction is not null:
            for each step in g4Correction.steps:
                tempA[step.gainStageKey] += step.delta
        eirpB = computeAllChannelEirp(channels, tempA)

        if NOT checkEirpWithinLimits(eirpB, initialEirp, channels, negLimit, posLimit):
            reject

        accept

function checkEirpWithinLimits(eirp, initialEirp, channels, negLimit, posLimit):
    for each channel in channels:
        dev = eirp[channel.id] - initialEirp[channel.id]
        if negLimit != null AND dev < -(negLimit + 0.001): return false
        if posLimit != null AND dev >  (posLimit + 0.001): return false
    return true

10.2 Pseudocode: Phase 2 EIRP Filter (Standard)

function filterEirpStandard(candidates, gainValues, channels, initialEirp, params):
    negLimit = params.maxNegativeEirpDeviation
    posLimit = params.maxPositiveEirpDeviation

    return filter candidates where:
        temp = copy of gainValues
        for each step in move.steps:
            temp[step.gainStageKey] += step.delta
        eirp = computeAllChannelEirp(channels, temp)
        checkEirpWithinLimits(eirp, initialEirp, channels, negLimit, posLimit) == true

11. G4 Correction Computation

REQ-070 The G4 correction SHALL compare the current EIRP of each channel with the initial EIRP and compute a G4 delta that opposes the deviation.
REQ-071 The G4 correction delta SHALL be quantized to the nearest whole number of G4 granularity steps.
REQ-072 If the quantized correction is zero steps for a channel, that channel SHALL NOT produce a G4 atomic step.
REQ-073 If no channels require correction (all quantized to zero), the G4 correction step SHALL be omitted entirely.

11.1 Formula

For each channel c:

deviationc = EIRPc(current) − EIRPc(initial) rawCorrectionc = −deviationc numStepsc = round(rawCorrectionc / γG4,c) ΔG4c = numStepsc · γG4,c

11.2 Pseudocode

function computeG4CorrectionAfter(gainValues, channels, initialEirp,
                                    granularities, gainStages):
    currentEirp = computeAllChannelEirp(channels, gainValues)
    return computeG4StepsFromDeviation(initialEirp, currentEirp,
                                        channels, gainValues, granularities, gainStages)

function computeG4StepsFromDeviation(referenceEirp, actualEirp,
                                      channels, gainValues, granularities, gainStages):
    g4Steps = []

    for each channel in channels:
        deviation = actualEirp[channel.id] - referenceEirp[channel.id]
        if |deviation| < 0.001: continue     // negligible

        g4Key = "G4:ch" + channel.id
        g4Gran = granularities[g4Key]
        if g4Gran is undefined: continue

        rawCorrection = -deviation
        numSteps = round(rawCorrection / g4Gran)
        if numSteps == 0: continue

        g4Steps.push({gainStageKey: g4Key, delta: numSteps * g4Gran})

    if g4Steps is empty: return null

    return {steps: g4Steps, stageType: "G4"}
Quantization residual. When the EIRP deviation is not an exact multiple of the G4 granularity, the correction will have a residual. For example, if the deviation is 0.37 dB and γG4 = 0.25 dB, the correction is round(−0.37 / 0.25) = −1 step = −0.25 dB, leaving a residual of −0.12 dB. This residual accumulates but is bounded by γG4/2.

12. Scoring & Selection

REQ-080 Among feasible candidates, the algorithm SHALL select the one with the most total gain progress (largest sum of absolute deltas).
REQ-081 When two candidates have equal progress (within 0.001 dB), the algorithm SHALL prefer the one with fewer atomic steps.

12.1 Formula

score(move) = − ∑s ∈ move.steps |s.delta|

Lower score = better (more progress). The candidate with the minimum score is selected.

12.2 Pseudocode

function pickBestByProgress(feasible):
    scored = []
    for each move in feasible:
        progress = sum of |step.delta| for step in move.steps
        scored.push({move, score: -progress})

    sort scored by:
        1. score ascending (more progress first)
        2. on tie (|scoreA - scoreB| < 0.001): fewer steps preferred

    return scored[0].move

13. Power Level Cascade

REQ-090 Power at each gain stage output SHALL be computed by cascading through the channel's gain chain in dB.
REQ-091 For shared stages (G1, G2, G6, G7), the output power SHALL be the sum in the linear domain of all channels passing through that stage, converted back to dBm.
REQ-092 For per-channel stages (G3, G4, G5), the output power SHALL be the single channel's cascaded power.

13.1 Formulas

Per-channel cascade (dB domain):

Pout(kj, c) = Pin,c + ∑i=1..j gki

Shared stage aggregation (linear domain):

Pshared(k) = 10 · log10 &Big( ∑c ∈ C(k) 10Pout(k,c)/10 &Big;)    [dBm]

where C(k) is the set of all channels passing through stage k.

13.2 Pseudocode

function computeAllPowerLevels(channels, gainValues):
    powerLevels = {}

    for each channel in channels:
        chain = getChannelGainChain(channel)
        pIn = channel.ipfd + 10 * log10(channel.bandwidthMHz * 1e6)
        cumulative = pIn

        for each key in chain:
            cumulative += gainValues[key]
            stageType = key.split(":")[0]

            if stageType in {"G3", "G4", "G5"}:
                // Per-channel: store directly
                powerLevels[key] = cumulative
            else:
                // Shared: accumulate in linear domain
                linear = 10^(cumulative / 10)
                if key in powerLevels:
                    existingLinear = 10^(powerLevels[key] / 10)
                    powerLevels[key] = 10 * log10(existingLinear + linear)
                else:
                    powerLevels[key] = cumulative

    return powerLevels

14. System Noise Temperature

REQ-100 The system noise temperature SHALL be computed at every step using the Friis cascade formula over the receive chain (G1, G2, G3 only).
REQ-101 Transmit-side stages (G4–G7) SHALL NOT contribute to the system noise temperature.

14.1 Formula

Tsys,c = Tant,c + TG1 + TG2 / G1lin + TG3 / (G1lin · G2lin)

where:

  • Tant,c = antenna noise temperature for channel c (K)
  • Tn = 290 · (10NFn/10 − 1)    [K] — equivalent noise temperature of stage n
  • Gnlin = 10gn/10 — linear gain of stage n
  • NFn = noise figure of stage n (dB)

14.2 Pseudocode

function computeChannelSystemTemp(channel, gainValues, gainStages):
    T0 = 290   // standard reference temperature (K)
    chain = getChannelGainChain(channel)
    rxStageKeys = filter chain to keys starting with "G1:", "G2:", "G3:"

    tSys = channel.antennaNoiseTemp
    cumulativeGainLinear = 1.0

    for each key in rxStageKeys:
        stage = gainStages.get(key)
        stageTemp = T0 * (10^(stage.noiseFigure / 10) - 1)
        stageGainLinear = 10^(gainValues[key] / 10)

        tSys += stageTemp / cumulativeGainLinear
        cumulativeGainLinear *= stageGainLinear

    return tSys

15. Deadlock Resolution

REQ-110 When no candidate passes strict power threshold checks, the algorithm SHALL relax all power thresholds by 1.0 dB in both directions and retry.
REQ-111 EIRP deviation limits SHALL NOT be relaxed during deadlock resolution.
REQ-112 Each threshold relaxation SHALL be counted and reported in thresholdViolations.
REQ-113 If no candidate passes even with relaxed thresholds, the algorithm SHALL terminate with converged = false.

15.1 Pseudocode

function filterRelaxedPower(candidates, gainValues, channels, gainStages):
    return filter candidates where:
        tempGains = copy of gainValues
        for each step in move.steps:
            tempGains[step.gainStageKey] += step.delta
        powerLevels = computeAllPowerLevels(channels, tempGains)
        for each (key, power) in powerLevels:
            stage = gainStages.get(key)
            if power > stage.upperThreshold + 1.0: reject
            if power < stage.lowerThreshold - 1.0: reject
        accept

16. Termination & Output

REQ-120 The algorithm SHALL terminate when any of the following conditions is met:
  1. All gain stages have reached their target values (converged = true)
  2. No feasible move exists, even after deadlock resolution (converged = false)
  3. EIRP limits are infeasible (error is set, converged = false)
  4. The iteration limit is reached (converged = false)
REQ-121 Convergence SHALL be checked with tolerance: |gk − gktarget| < 0.01 · γk for all k.
REQ-122 The output SHALL include a complete ordered list of TransitionStep records, each containing a full system state snapshot (gain values, EIRP, power levels, system temperature).
REQ-123 When EIRP limits are infeasible, the error message SHALL include the smallest achievable worst-case deviation among all power-feasible candidates. This is computed by checking the intermediate (post-primary, pre-G4) state of each power-feasible candidate.
